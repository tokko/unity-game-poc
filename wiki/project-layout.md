# Project Layout

<!-- 2026-05-07 -->

## Authoritative path

`D:\claude projects\unity-game-poc` — this is the intended permanent home.

## Why two paths exist

When Unity first created the project it used `C:\Temp\unity-game-poc` (the Hub GUI default),
because `D:\claude projects\unity-game-poc` was locked by an unknown process (Windows Error 32
sharing violation on the directory itself, not its contents). Files can be created inside the
locked directory, but the directory itself cannot be renamed or deleted.

**Current state**: the Unity Editor still opens the project from `C:\Temp\unity-game-poc`.
The Unity Hub registry (`%APPDATA%\UnityHub\projects-v1.json`) was updated to point to
`D:\claude projects\unity-game-poc`, but the Editor session keeps the temp path active until
it is closed and reopened from the Hub.

## Sync strategy

Use robocopy to copy from the temp path to the target, excluding `Library/` and `Temp/`
(both are regenerated by Unity on open):

```powershell
robocopy "C:\Temp\unity-game-poc" "D:\claude projects\unity-game-poc" /E /XD "Library" "Temp"
```

Scripts written directly to `D:\claude projects\unity-game-poc\Assets\` must also be copied
to `C:\Temp\unity-game-poc\Assets\` for the currently running Editor to see them.
