# Session History — May 2026

## 2026-05-07 — Initial build + wiki setup

### What was done
Bootstrapped the entire project from scratch on a fresh Windows 11 machine. Installed Unity Hub
(winget hash-mismatch workaround required — downloaded installer directly) and Unity 6 LTS
(6000.0.74f1) via the Hub GUI. Created a URP "Universal 3D" project. Wrote three C# components:

- **PlayerController** — Rigidbody WASD movement with camera-relative direction, Space jump,
  `Physics.Raycast` ground check. Key Unity 6 API: `rb.linearVelocity` (renamed from `.velocity`).
- **CameraController** — scroll-wheel-blended top-down ↔ third-person camera in `LateUpdate`.
- **SceneAutoSetup** — editor-only `[InitializeOnLoad]` script that auto-creates GameScene
  (Ground plane, Player cylinder, Camera, Directional Light) on first compile.

### Bugs found and fixed
1. **Scripts not visible in Editor**: Unity was running from `C:\Temp\unity-game-poc` (Hub default)
   but scripts were written to `D:\claude projects\unity-game-poc`. Fixed by copying to both paths.
2. **SceneAutoSetup crashed during play mode**: `EditorSceneManager.NewScene()` throws if
   `delayCall` fires while Editor is in play mode. Workaround: exit play mode, use
   `Tools > Re-run Scene Setup`.
3. **Input system conflict (9994 errors)**: URP template defaults to New Input System only;
   `UnityEngine.Input` throws at runtime. Fixed by setting Active Input Handling → "Both".
4. **Jump never triggered**: `groundCheckDistance = 0.6f` couldn't reach the ground plane at y=0
   from the cylinder center at y=1 (needs >1.05). Fixed to `1.1f` in script and Inspector.

### Decisions
- Keeping legacy `UnityEngine.Input` with "Both" active input handling for simplicity on Personal plan.
- Ground check uses a downward raycast from center+0.05 rather than `OnCollisionStay` to avoid
  edge-case false positives on sloped geometry.
